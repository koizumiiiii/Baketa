# Ultrathink分析

複雑な問題に対して深い分析を実行します。

## 分析フレームワーク

### 1. 根本原因分析
```
- 問題の本質は何か？
- 表面的な症状と根本原因を区別
- 「なぜ？」を5回繰り返す
```

### 2. 影響範囲調査
```
- 変更対象ファイルの特定
- 依存関係の確認（参照元/参照先）
- テストへの影響
- パフォーマンスへの影響
```

### 3. 解決策の検討
```
- 複数の選択肢を列挙
- 各選択肢のメリット/デメリット
- 実装コストと効果のバランス
```

### 4. 実装計画
```
- フェーズ分け（必要な場合）
- リスク軽減策
- 検証方法
```

## 出力フォーマット

```markdown
## Ultrathink分析: <タイトル>

### 1. 問題の本質
<根本原因の説明>

### 2. 影響範囲
| 対象 | 影響 |
|-----|------|
| file1.cs | 変更必要 |

### 3. 解決策オプション
| オプション | メリット | デメリット |
|-----------|---------|-----------|
| A案 | ... | ... |
| B案 | ... | ... |

### 4. 推奨方針
<選択した方針とその理由>

### 5. 実装計画
1. Phase 1: ...
2. Phase 2: ...
```
