# Issue作成

GitHub Issueを標準テンプレートで作成します。

## Ultrathink分析フロー

1. **要件整理**: ユーザーの要求を明確化
2. **影響範囲調査**: 関連ファイル、依存関係を特定
3. **実装方針検討**: 複数の選択肢を比較
4. **Geminiレビュー**: 方針についてフィードバック取得

## Issue作成コマンド

### バグ報告
```bash
gh issue create --title "fix: <簡潔な説明>" --body "<本文>" --label "bug"
```

### 機能追加
```bash
gh issue create --title "feat: <簡潔な説明>" --body "<本文>" --label "enhancement"
```

## Issueテンプレート

```markdown
## 概要
<1-2文で概要を説明>

## 背景/現状の問題
<なぜこの変更が必要か>

## 要件
- [ ] 要件1
- [ ] 要件2

## 実装方針
<技術的なアプローチ>

## 影響範囲
| ファイル | 変更内容 |
|---------|---------|
| file1.cs | 変更1 |

## チェックリスト
- [ ] 実装完了
- [ ] テスト追加
- [ ] ビルド成功確認
```

## 注意事項
- Issueタイトルは `fix:`, `feat:`, `chore:` などのプレフィックスを付ける
- 関連Issueがあれば `## 関連` セクションにリンクを追加
